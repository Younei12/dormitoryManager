在 学生管理 模块中，显示 student 表中所有学生的信息
    可按学号或者按姓名查找学生          //  单独做一个组件，对于学生的查询都是用这个组件
    可对学生信息进行编辑（所有操作，包括修改和分配宿舍操作）
        改变学生性别，清空 楼栋和宿舍号信息，住宿状态改为未分配
    可添加学生信息（单个添加和批量添加）

在 分配宿舍 模块中，仅显示未分配宿舍的学生的信息
    可按学号或者按姓名查找学生
    仅能对学生进行分配宿舍操作，        // 单独组一个组件，宿舍的分配和更换使用同一个组件
        先选楼栋，性别为男禁选女寝，性别为女禁选男寝
        再选宿舍号，显示所选楼栋中 未达到总容量 的宿舍号，保存后 改宿舍的已住人数++，剩余床位--

在 更换宿舍 模块中，仅显示已分配宿舍的学生的信息
    可按学号或者按姓名查找学生
    仅能对学生进行分配宿舍操作
        先选楼栋，性别为男禁选女寝，性别为女禁选男寝
        再选宿舍号，显示所选楼栋中 未达到总容纳量 的宿舍号

在 办理退宿 模块中，仅显示已分配宿舍的学生的信息
     input:学号或者姓名
     可对学生的住宿状态进行修改，由住宿修改为-->退宿，修改的同时，改变 学生信息表 中的住宿状态字段
     设置一个按钮，开启则为住宿（绿色），关闭则为退宿（红色）     // 单纯一个按钮控制状态是否合理？
     是否对退宿学生宿舍的公共财物进行登记？  // 嫌麻烦，但是单独一个按钮又觉得太简单

    (退宿成功一个，则 该学生的 住宿状态=3 楼栋和宿舍字段清空，宿舍表中该宿舍的 在住人数--，剩余床位++)

在 楼栋管理 模块中，渲染 宿舍楼栋表 中所有的楼栋信息
    输入框：楼栋编号/名称 搜索，select ：男寝/女寝
    楼栋的增删改查

在 宿舍管理 模块中，渲染 宿舍表 中所有寝室的信息
    搜索框：宿舍号搜，select:不同楼栋
    tr>td:寝室号--楼栋--总容纳人数--已住人数--剩余床位
    添加宿舍：  选择楼栋--设置宿舍号--设置容纳人数

    搜索框：学生姓名，select:维修状态
    给未分配的维修员的保修单分配维修员
    为保留维修记录，不对维修单进行删除操作
